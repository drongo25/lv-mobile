# Техническое задание: Система управления сервисной службой (FMS)

## 1. Общие сведения
Система предназначена для автоматизации работы компании, занимающейся обслуживанием удаленного оборудования. Цель — обеспечить непрерывную работу объектов, прозрачность действий персонала и точность прогнозирования ресурсов.

## 2. Объекты обслуживания и процессы
Оборудование распределено территориально (город и область). Для каждого объекта зафиксированы GPS-координаты.

**Регламентные работы включают:**
* **Заправка:** доставка и залив топлива (бензин/дизель).
* **ТО-1:** проверка состояния и замена фильтрующих элементов.
* **ТО-2:** контроль уровня моторного масла и его доливка/замена.
* **Учет наработки:** фиксация показаний счетчика моточасов.

## 3. Функциональные требования к системе

### 3.1. Аналитика и прогнозирование
* **Учет топлива:** Мониторинг текущего остатка в баках и расчет прогнозного времени работы до критической отметки (остановка двигателя).
* **Планирование закупок:** Прогноз необходимого объема топлива на месяц на основе среднего расхода каждой единицы оборудования.
* **График ТО:** Автоматическое уведомление о необходимости замены масла и фильтров при достижении определенного количества моточасов.

### 3.2. Управление задачами (Диспетчеризация)
* Формирование ежедневных заданий для операторов.
* Возможность передачи активной задачи от одного оператора другому в случае поломки автомобиля или смены графика.
* Приоритезация задач (экстренная дозаправка vs плановое ТО).

### 3.3. Контроль и отчетность
Для подтверждения факта выполнения работ оператор передает данные через мобильный интерфейс:
1.  **Геолокация:** Автоматическая фиксация координат в момент начала и окончания работ.
2.  **Фотофиксация:**
    * Общий вид оборудования до начала работ.
    * Фото новых и замененных (отработанных) фильтров.
    * Фото приборной панели с показаниями моточасов.
3.  **Дополнительный контроль:**
    * *Сверка «Норма/Факт»:* сравнение фактического объема залитого топлива с расчетным расходом по моточасам.
    * *Чек-лист:* обязательное заполнение пунктов о чистоте площадки и отсутствии подтеков масла.

## 4. Интерфейс взаимодействия (Telegram-бот)

### 4.1. Профиль Оператора
* Получение списка задач на день с кнопками навигации.
* Инструменты отчетности (загрузка фото, ввод цифр, отправка геопозиции).
* Сводка по выполненным и оставшимся заданиям в режиме реального времени.

### 4.2. Профиль Руководителя
* **Управление доступом:** Добавление новых сотрудников, блокировка учетных записей (при увольнении сотруднику закрывается доступ к боту и истории данных).
* **Мониторинг:** Просмотр общего статуса сети оборудования.
* **Логирование:** Полный аудит действий (кто, когда и какую задачу взял, какие данные изменил). Все изменения в базе данных записываются в системный лог.

## 5. Информационная безопасность
* Авторизация в боте по номеру телефона или уникальному ID.
* Хранение истории перемещений и отчетов на защищенном сервере.
* Запрет на редактирование отчета после его закрытия.













# Техническое задание (расширенная системная спецификация)

## FMS — Fuel & Field Maintenance System

---

# 1. Общие сведения

## 1.1. Наименование системы

**FMS (Fuel & Field Maintenance System)** — система управления сервисным обслуживанием распределенного оборудования.

---

## 1.2. Назначение системы

Система предназначена для:

* управления сервисной службой;
* контроля выполнения регламентных работ;
* мониторинга состояния оборудования;
* учета топлива и моточасов;
* диспетчеризации операторов;
* формирования отчетности;
* предотвращения мошенничества;
* прогнозирования потребления топлива;
* ведения полного аудита действий персонала.

---

## 1.3. Цели внедрения

В результате внедрения система должна обеспечить:

* снижение простоев оборудования;
* прозрачность работы операторов;
* сокращение потерь топлива;
* автоматизацию диспетчеризации;
* прогнозирование закупок топлива;
* контроль качества обслуживания;
* централизованное хранение данных.

---

# 2. Архитектура системы

## 2.1. Компоненты системы

Система состоит из:

### Серверная часть (Backend)

Обеспечивает:

* бизнес-логику;
* API;
* обработку задач;
* хранение данных;
* аналитику;
* авторизацию;
* журналирование.

---

### Telegram Bot Interface

Интерфейс для:

* операторов;
* диспетчеров;
* руководителей.

---

### Web Admin Panel

Интерфейс руководителя для:

* мониторинга;
* управления пользователями;
* просмотра аналитики;
* формирования отчетов.

---

### Аналитический модуль

Отвечает за:

* прогноз топлива;
* расчет моточасов;
* выявление аномалий;
* SLA-контроль.

---

### Геомодуль

Обеспечивает:

* работу с GPS;
* построение маршрутов;
* контроль прибытия;
* геоаналитику.

# 3. Объекты системы

---

# 3.1. Оборудование

Для каждого объекта хранится:

| Поле                    | Описание                  |
| ----------------------- | ------------------------- |
| ID                      | Уникальный идентификатор  |
| Инвентарный номер       | Внутренний номер          |
| Серийный номер          | Производственный номер    |

| Тип топлива             | Дизель / бензин           |

| Статус                  | Активен / ремонт / авария |

# 7. Контроль выполнения работ

Система должна:

* сравнивать расстояние до объекта;
* запрещать закрытие задачи вне радиуса.
